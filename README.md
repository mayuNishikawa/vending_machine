## クラス設計をする上でで気をつけたこと
+ データとそれに関わる処理の関係性がシンプルでわかりやすいようにクラス設計した
+ 単一責任の原則を意識してクラスを考えた
+ 図を使って可視化して、モノの関係性や動きを意識しながら設計した
+ 見やすい, 修正しやすい, ひと目見て何をしているか分かるメソッド定義及び命名
+ あえて、drinkではなくitem、moneyではなくcasherなど抽象性の高い命名を意識(コードの汎用性を意識)
